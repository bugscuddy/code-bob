# AI-Powered Website Builder Mobile App

An Android mobile app that allows users to describe websites via text prompts, which are then generated by AI using Ollama, with a FastAPI backend handling authentication, AI processing, and file exports.

## Tech Stack

### Client (Expo SDK 54)
- React Native with Expo
- React Navigation
- React Native Paper (UI components)
- Zustand (state management)
- Axios (API client)
- WebView (website preview)

### Server (FastAPI)
- Python 3.11+
- FastAPI framework
- SQLAlchemy (ORM)
- SQLite (development database)
- JWT authentication
- Ollama (AI integration)

## Prerequisites

- Node.js and Bun (for client)
- Python 3.11+ (for server)
- Ollama installed and running
- Android Studio / Android Emulator (for development)

## Setup Instructions

### 1. Install Ollama

Download and install Ollama from https://ollama.com

Pull a model for website generation:
```bash
ollama pull llama3:8b
```

Start Ollama:
```bash
ollama serve
```

### 2. Server Setup

Navigate to the server directory:
```bash
cd server
```

Create a virtual environment (if not already created):
```bash
python3 -m venv .venv
```

Activate the virtual environment:
```bash
source .venv/bin/activate
```

Install dependencies:
```bash
pip install -r requirements.txt
```

Create environment file:
```bash
cp .env.example .env
```

Edit `.env` with your configuration:
```env
DATABASE_URL=sqlite:///./website_builder.db
SECRET_KEY=your-secret-key-change-this-in-production
OLLAMA_MODEL=llama3:8b
OLLAMA_BASE_URL=http://localhost:11434
```

Seed the database with templates:
```bash
python seed_templates.py
```

Start the server:
```bash
uvicorn app.main:app --reload --host 0.0.0.0 --port 8000
```

The API will be available at http://localhost:8000

### 3. Client Setup

Navigate to the client directory:
```bash
cd client
```

Install dependencies with Bun:
```bash
bun install
```

The `.env` file should already be configured with:
```env
EXPO_PUBLIC_API_URL=http://localhost:8000
```

If running on a physical device, update this to your machine's local IP address.

Start the Expo development server:
```bash
bun start
```

Run on Android:
```bash
bun run android
```

## Usage

1. **Register/Login**: Create an account or login with existing credentials
2. **Generate Website**: 
   - Navigate to the "Generate" tab
   - Enter a description of the website you want
   - Optionally select a template
   - Choose a style (modern, minimal, corporate, etc.)
   - Optionally specify an industry
   - Click "Generate Website"
3. **Preview**: View the generated website in the built-in WebView
4. **Export**: Download the HTML/CSS/JS files as a ZIP
5. **Projects**: View and manage your generated websites
6. **Templates**: Browse available templates

## API Endpoints

### Authentication
- POST `/api/auth/register` - Register a new user
- POST `/api/auth/login` - Login and get JWT token
- GET `/api/auth/me` - Get current user info

### Projects
- POST `/api/projects/generate` - Generate a website from prompt
- GET `/api/projects/` - List user's projects
- GET `/api/projects/{id}` - Get project details
- DELETE `/api/projects/{id}` - Delete a project
- GET `/api/projects/{id}/export` - Export project as ZIP

### Templates
- GET `/api/templates/` - List all templates
- GET `/api/templates/{id}` - Get template details
- POST `/api/templates/` - Create a new template (admin)

## Features

- **AI-Powered Generation**: Uses Ollama (llama3) to generate complete websites from text descriptions
- **Template Library**: Pre-built templates for common website types
- **Style Customization**: Choose from modern, minimal, corporate, creative, and dark styles
- **Industry-Specific Generation**: Specify industry for better-tailored results
- **Real-time Preview**: Preview generated websites directly in the app
- **Export Functionality**: Download complete HTML/CSS/JS files

- **Project Management**: Save, view, and delete generated websites

## Troubleshooting

### Ollama Connection Issues
- Ensure Ollama is running: `ollama serve`
- Check Ollama is accessible at http://localhost:11434
- Verify the model is pulled: `ollama list`

### Client API Connection Issues
- Ensure the server is running on port 8000
- Check the EXPO_PUBLIC_API_URL in client/.env
- For physical devices, use your machine's local IP address instead of localhost

### Build Errors
- Clear cache: `bun start --clear`
- Reinstall dependencies: `rm -rf node_modules && bun install`
- Check Expo SDK version compatibility
